# Design Reference Notes

## pingdotgg/t3code

- Dense product shell with persistent navigation and utility surfaces.
- Small controls, compact rows, and clear state indicators.
- Strong agent documentation through `AGENTS.md`.
- Monorepo boundaries with app/package separation.

## devxsubh/rtp-screener

- Product-first authenticated experience rather than a marketing front door.
- Shadcn-like primitives, restrained radius, and strong form/button affordances.
- Backend auth and route protection patterns.
- Compliance/audit language that keeps human review in the loop.

## Portal Translation

- Use a left navigation shell, top command bar, compact KPI strip, tables, queues, and audit timeline.
- Use neutral surfaces with blue action accents and semantic status colors.
- Do not copy reference code or brand identity.
- Make role, scope, approvals, and auditability visible in the first viewport.
